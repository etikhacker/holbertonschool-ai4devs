# User Stories

### User Story 1
As a student, I want to view all my upcoming assignments in one place so that I can plan my study schedule effectively.

**Acceptance Criteria**:
- Student can see a list of all assignments sorted by due date.
- Each assignment displays the subject, title, and remaining days until deadline.
- Overdue assignments are highlighted in red.

**Priority**: MVP

---

### User Story 2
As a student, I want to receive reminders before assignment deadlines so that I never miss a submission.

**Acceptance Criteria**:
- Student receives a notification 24 hours before each deadline.
- Notifications are delivered via email or in-app alert.
- Student can customize reminder timing in settings.

**Priority**: MVP

---

### User Story 3
As a student, I want to calculate my current GPA so that I can track my overall academic performance.

**Acceptance Criteria**:
- Student enters grades and credit hours for each course.
- System calculates and displays the current GPA instantly.
- GPA history is shown as a visual chart over time.

**Priority**: MVP

---

### User Story 4
As a teacher, I want to create and distribute assignments to my class so that students receive clear instructions and deadlines.

**Acceptance Criteria**:
- Teacher can create an assignment with title, description, and due date.
- Assignment is instantly visible to all students in the selected class.
- Teacher receives confirmation that the assignment was published.

**Priority**: MVP

---

### User Story 5
As a teacher, I want to view the progress of each student in my class so that I can identify who needs additional support.

**Acceptance Criteria**:
- Teacher sees a dashboard listing all students with their current grades.
- Students below a defined threshold are flagged automatically.
- Teacher can click on any student to view detailed performance data.

**Priority**: High

---

### User Story 6
As a teacher, I want to track student attendance so that I can monitor participation and report absences accurately.

**Acceptance Criteria**:
- Teacher can mark students as present, absent, or late for each session.
- Attendance records are saved and viewable by date.
- System generates a summary report of absences per student.

**Priority**: High

---

### User Story 7
As a student, I want to receive an AI-generated study plan so that I can focus on subjects where I need the most improvement.

**Acceptance Criteria**:
- AI analyzes the student's grades and identifies weak subjects.
- A personalized weekly study plan is generated automatically.
- Student can accept, modify, or regenerate the study plan.

**Priority**: Medium

---

### User Story 8
As a student, I want to switch between light and dark mode so that I can use the platform comfortably in any lighting condition.

**Acceptance Criteria**:
- Student can toggle between light and dark mode from the settings menu.
- The selected theme is saved and applied automatically on next login.
- All pages and components render correctly in both modes.

**Priority**: Low
